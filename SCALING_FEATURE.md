# 复核界面自适应缩放功能说明

## 🎯 问题解决

之前复核界面在打开大尺寸图片时，会超出窗口范围，导致无法完整查看。现已添加自适应缩放功能，可自动调整图片尺寸以适配窗口。

## ✨ 新增功能

### 1. **自动适应窗口**
- 打开图片时自动缩放至适合窗口显示
- 无需手动调整

### 2. **缩放工具栏**
在画布顶部添加了缩放控制按钮：
- **−** (减小)：缩小图片 25%
- **适应**：自动缩放至窗口大小（推荐用于大图）
- **100%**：恢复原始尺寸
- **+** (增大)：放大图片 25%
- **显示标签**：实时显示当前缩放百分比

### 3. **键盘快捷键**
| 快捷键     | 功能       |
| ---------- | ---------- |
| `+` 或 `=` | 放大       |
| `-`        | 缩小       |
| `Shift+0`  | 自适应窗口 |

### 4. **平滑缩放**
- 缩放时自动调整所有标注框位置和大小
- 标签字体也会按比例调整
- 缩放不影响保存的原始坐标（自动处理）

## 🔄 工作原理

### 坐标系统
```
原始图片坐标
    ↓
预览图坐标 (通过 _reviewScaleX / _reviewScaleY)
    ↓
Canvas 显示坐标 (通过 _canvasDisplayScale)
```

### 保存流程
1. 用户在缩放后的 Canvas 上编辑标注框
2. 点击「保存」时，系统自动反算坐标：
   - Canvas 显示坐标 ÷ _canvasDisplayScale → 预览图坐标
   - 预览图坐标 ÷ _reviewScaleX/Y → 原始图片坐标
3. 保存到数据库的始终是原始图片坐标（与缩放无关）

## 📝 使用示例

### 场景 1：打开大图片
```
用户打开 1920×2560 的原始图片
    ↓
系统自动检测窗口大小
    ↓
自动缩放至 50% (或其他合适比例)
    ↓
图片完整显示在窗口内 ✓
```

### 场景 2：进一步放大查看细节
```
用户点击 [+] 按钮
    ↓
当前缩放 50% → 125% × 50% = 62.5%
    ↓
所有标注框同步放大
    ↓
可以看到更清晰的细节
```

### 场景 3：编辑和保存
```
用户在缩放 75% 的 Canvas 上修改标注框
    ↓
系统计算：
- 编辑后的 box = [100, 50, 300, 80]（显示坐标）
- 保存前反算：[100÷75%, 50÷75%, 300÷75%, 80÷75%]
- 得到原始坐标（不受缩放影响） ✓
```

## 🛠️ 技术细节

### 修改的文件
- `templates/review_editor.html` - HTML、CSS 和 JavaScript 都在同一文件

### 新增全局变量
```javascript
let _canvasDisplayScale = 1;  // 缩放比例（1.0 = 100%）
```

### 新增函数
```javascript
updateCanvasScale(newScale)    // 更新缩放并调整所有对象
scaleCanvasToFit()             // 自动适应窗口
scaleCanvasUp()                // 放大 25%
scaleCanvasDown()              // 缩小 25%
scaleCanvasToActual()          // 恢复 100%
```

### 修改的坐标转换函数
- `boxOriginalToDisplay()` - 加载标注时考虑缩放
- `boxDisplayToOriginal()` - 保存标注时反缩放
- `posOriginalToDisplay()` - 加载文本位置时考虑缩放
- `posDisplayToOriginal()` - 保存文本位置时反缩放

## ⚙️ 缩放算法

```javascript
// 计算适应比例
scale = Math.min(
    (窗口宽度 - 内边距) / 图片宽度,
    (窗口高度 - 工具栏高度 - 内边距) / 图片高度,
    1.0  // 不超过 100%
)

// 限制范围：10% ~ 300%
scale = Math.max(0.1, Math.min(3, scale))
```

## 📊 性能考虑

- 缩放时不重新加载图片，仅调整 Canvas 尺寸和对象位置
- 使用 `requestAnimationFrame` 进行平滑更新
- 缩放时会标记"未保存"状态，提示用户需要保存

## 🐛 已知限制

1. **超大缩放**：缩放到 10% 时可能很难精确编辑
   - 解决：鼠标悬停时可以看到坐标显示，或按 `+` 放大

2. **多次快速缩放**：很快点击 `+` 多次可能导致轻微延迟
   - 解决：系统会自动调和缩放速率（不会超过 300%）

3. **导出坐标精度**：极限缩放下坐标可能有 ±1 像素偏差
   - 原因：四舍五入误差级联
   - 影响：通常不可见
