# 人才信息解析系统（Web 版）

一个面向“人才材料/证明文件”的 **智能解析 + 可视化复核** 平台。

- **上传材料**（PDF/图片，支持中文文件名）
- **调用大模型** 进行字段抽取与结构化输出
- **生成可复核的标注图片 + JSON**
- 在「数据复核」页面进行 **框选/改字段/保存/导出**

---

## 适用场景

- 人才申报材料整理：身份证、学历学位、工作证明、项目证明、论文/著作/专利/奖项等
- 需要把“文件里的关键信息”变成可检索、可复核的结构化数据
- 需要“可解释”的输出：每个字段都有对应的 **box 坐标**，支持人工复核

---

## 功能一览

- **智能解析**：上传文件后自动解析并产出结构化结果
- **数据复核**：在浏览器里查看原图并编辑标注（拖拽框、修改字段、保存）
- **导出**：支持导出带标注的图片/打包下载（视页面功能而定）
- **安全上传**：保留中文文件名，同时防路径穿越/非法字符
- **性能优化**：复核页默认加载 `*.preview.jpg` 预览图，减少大图渲染卡顿；保存仍以原图坐标为准

---

## 项目结构（核心）

```
app.py                # Flask 入口：路由、上传、复核 API、权限
models.py             # 用户/权限等数据模型（SQLAlchemy）
utils.py              # 解析流程、PDF 转图、LLM 调用、产物写入、预览图生成
templates/            # 前端页面（Jinja2）
static/               # CSS/JS 静态资源
prompt/               # 解析提示词（*.txt）
uploads/              # 用户上传与解析产物（运行数据）
requirements.txt      # 依赖
```

## 快速开始（本地）

1. 激活 conda 环境

```bash
conda activate aliyun
```

2. 安装依赖（建议使用虚拟环境）

```bash
pip install -r requirements.txt
```

3. 启动

```bash
python app.py
```

3. 访问

- 首页：`/TIE/`
- 智能解析：`/TIE/intelligent-analysis`
- 数据复核：`/TIE/data-review`

> 说明：本项目路由挂载在 `/TIE` 前缀下。

---

## 运行数据说明

解析后的产物会落在：

```
uploads/<user_id>/
  xxx_1.jpg            # 解析产物图片（原尺寸）
  xxx_1.preview.jpg    # 复核用预览图（长边≤1600）
  xxx_1.json           # 解析结果（含 box 等）
  xxx_1.review.json    # 复核保存结果
```

复核页默认使用预览图显示，并自动按比例换算标注坐标；保存时会回写为原图坐标，保证导出/二次处理一致性。

## 部署提示（服务器）

若服务器上出现“本地正常、服务器 500”，常见原因包括：

- 启动工作目录不同导致相对路径失效（本项目已将 `prompt/` 读取改为绝对路径）
- 服务器缺少图像/PDF 依赖或权限不足
- OpenCV 在部分环境下写中文路径不稳定（本项目已改为 `imencode + open(..., 'wb')`）
